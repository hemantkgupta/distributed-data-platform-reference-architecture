## Goal

Day 6 focuses on contract-driven medallion promotion: bronze to silver to gold, with validation evidence at each boundary.

The goal is to make promotion explicit:

- Bronze preserves source-fidelity events, including duplicates and bad inputs.
- Silver publishes clean current-order rows only after blocker checks pass.
- Gold publishes daily business metrics only after reconciliation checks pass.
- Promotion evidence explains why a candidate was safe to expose.

## What I Learned

Medallion is not only folder naming. It is a sequence of contract boundaries.

Bronze is allowed to be messy because it preserves source truth. But it should not be unexplained: rows need source ids, offsets, contract metadata, and ingestion time.

Silver changes both quality and grain. In this lab, silver deduplicates event deliveries, removes invalid rows, and publishes one current row per order. That means silver is not just a filtered bronze table; it is a new semantic surface.

Gold is consumer-facing. It intentionally loses detail to publish daily business metrics. That makes reconciliation important: the gold revenue total must match the silver current-order source of truth.

The key operational pattern:

```text
candidate table -> promotion evidence -> promotion decision -> published table
```

Principal-level point: `CREATE TABLE AS SELECT` is compute. Promotion is governance plus quality evidence plus visibility control.

## PySpark Notes

The notebook uses PySpark for two transformations:

1. Bronze validation:

```python
validated_bronze_df = (
    bronze_df
    .withColumn(
        "dq_error",
        F.concat_ws(
            "; ",
            F.when(F.col("order_id").isNull(), F.lit("missing order_id")),
            F.when(F.col("amount_after") < 0, F.lit("negative amount_after")),
        ),
    )
)
```

SQL equivalent:

```sql
SELECT *,
  CONCAT_WS('; ',
    CASE WHEN order_id IS NULL THEN 'missing order_id' END,
    CASE WHEN amount_after < 0 THEN 'negative amount_after' END
  ) AS dq_error
FROM orders_bronze_day6;
```

2. Silver current-state derivation:

```python
silver_candidate_df = (
    valid_bronze_events_df
    .withColumn("event_rn", F.row_number().over(dedup_event_window))
    .where(F.col("event_rn") == 1)
    .withColumn("order_rn", F.row_number().over(latest_order_window))
    .where(F.col("order_rn") == 1)
    .select(...)
)
```

SQL equivalent:

```sql
SELECT *
FROM (
  SELECT *,
    ROW_NUMBER() OVER (PARTITION BY event_id ORDER BY ingested_at DESC) AS event_rn,
    ROW_NUMBER() OVER (PARTITION BY order_id ORDER BY source_lsn DESC, ingested_at DESC) AS order_rn
  FROM valid_bronze_events_day6
)
WHERE event_rn = 1 AND order_rn = 1;
```

Important syntax:

- `F.when(...)` is SQL `CASE WHEN`.
- `F.concat_ws(...)` joins multiple error strings into one validation column.
- `Window.partitionBy(...)` creates per-key calculation groups.
- `F.row_number().over(...)` is SQL `ROW_NUMBER() OVER (...)`.
- `createOrReplaceTempView(...)` bridges PySpark results back into SQL cells.

## Mistakes / Confusions

- Bronze being raw does not mean bronze is low-value. It is the replay and evidence layer.
- Silver is not just "cleaned bronze"; it may change grain and semantics.
- Gold should not be trusted only because it is aggregated. It needs reconciliation to its source of truth.
- Delta history explains table operations, but promotion evidence explains why a table was safe to expose.
- Warnings and blockers should be separate. A warning can be publishable; a blocker cannot.

## Revision Cards

Q: What does bronze own?
A: Source-fidelity records, replay metadata, raw contract metadata, and bad input evidence.

Q: What does silver own?
A: Cleaned, deduplicated, conformed, semantically stable records at a declared grain.

Q: What does gold own?
A: Consumer-ready business metrics, marts, and aggregates with quality and governance expectations.

Q: Why does silver need promotion evidence?
A: Because it changes quality and often grain. Consumers need proof that invalid rows, duplicates, and key constraints were handled.

Q: Why does gold need reconciliation?
A: Gold is usually aggregated and lossy. Reconciliation proves the metric still matches the trusted silver source.

Q: What is the difference between Delta history and promotion evidence?
A: Delta history records table operations. Promotion evidence records validation checks and publication decisions.
