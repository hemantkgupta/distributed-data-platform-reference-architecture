## What I Learned

CDC needs an envelope because the row alone is not enough to operate the pipeline.

The envelope needs:

- Contract identity: which contract version shaped this event.
- Source position: where replay resumes and how duplicates are identified.
- Operation type: create, update, delete, or snapshot.
- Before/after images: especially important for deletes and corrections.
- Aggregate key: how downstream sinks deduplicate or merge.
- Occurred/captured timestamps: separate business/source time from ingestion time.

An idempotency key should not be only the business key. A business key such as `order_id=42` can update many times. The idempotency key needs source position and event identity so replayed deliveries can be distinguished from new changes to the same entity.

## Mistakes / Confusions

- A CDC event is easy to confuse with the latest row state. It is actually a change record plus replay metadata.
- Deletes are invisible unless the envelope carries delete semantics and enough before-image data.
- At-least-once CDC means duplicates are normal. Idempotent sinks are not optional.

## Revision Cards

Q: Why is a CDC envelope more than a map of column names to values?
A: The envelope carries operation type, contract version, source offset, timestamps, and before/after images. Those fields make replay and debugging possible.

Q: Why do deletes need a before image?
A: After a delete, the row no longer exists. The before image preserves the key and useful old values so downstream tables can tombstone or retract correctly.

Q: Why is source offset part of the idempotency key?
A: CDC connectors can re-emit events after a crash. The sink needs to recognize the same source-positioned event instead of inserting a duplicate.

Q: What is the honest CDC delivery contract?
A: Usually at-least-once delivery from source log to broker, plus idempotent consumers and sinks.

Q: Where does this fit in medallion architecture?
A: Bronze should preserve source-fidelity events and replay metadata. Silver can deduplicate, conform, and apply business semantics.
